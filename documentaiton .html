Intégrez en Node.js l’envoi de vos SMS en Tunisie dans vos site web, vos applications et toutes vos solutions par API SMS

Intégrer l’API WinSMS pour envoyer des SMS en Tunisie n’a jamais été aussi simple. Grâce à notre API intuitive et facile à utiliser, vous pouvez envoyer des SMS d’alertes, consulter l’état de vos envoi SMS et plus encore, le tout en quelques lignes de code Node.js. Dans cette section, découvrez comment effectuer une intégration de base pour envoyer des SMS, que ce soit individuellement ou en masse, tout en gérant les attentes de réponse du serveur selon vos besoins.

1- Envoi d’un SMS à un seul numéro sans attente de la réponse du serveur en Node.js

Envoyez un SMS en Node.js rapidement sans attendre de réponse du serveur avec un exemple simple qui définit un timeout court. Idéal pour les messages urgents où la vitesse est essentielle.

NOS PRIX ET NOS PLANS

const https = require('https');
const querystring = require('querystring');

// Configuration des paramètres pour l'envoi du SMS en Tunisie avec WinSMS
const postData = querystring.stringify({
  'action': 'send-sms',
  'api_key': 'votre_clé_api', // Remplacez par votre clé API SMS Tunisie WinSMS réelle
  'to': '216xxxxxxxx', // Numéro de téléphone destinataire en Tunisie (Avec indicatif 216)
  'sms': 'Votre message ici', // Le message à envoyer
  'from': 'Votre_Sender_ID' // Votre identifiant d'expéditeur
});

const options = {
  hostname: 'www.winsmspro.com',
  port: 443,
  path: '/sms/sms/api',
  method: 'GET',
  headers: {
       'Content-Type': 'application/x-www-form-urlencoded',
       'Content-Length': postData.length
  }
};

// Initialisation de la requête pour l'envoi du SMS
const req = https.request(options, (res) => {
  console.log(`Statut de l'envoi : ${res.statusCode}`);

  res.on('data', (d) => {
    process.stdout.write(d);
  });
});

req.on('error', (e) => {
  console.error(`Erreur lors de l'envoi du SMS : ${e.message}`);
});

// Envoi de la requête avec les paramètres
req.write(postData);
req.end();

Ce code utilise les modules https et querystring de Node.js pour envoyer une requête HTTPS avec les paramètres nécessaires pour l’envoi d’un SMS via l’API WinSMS.

Astuce pour les développeurs : Vous pouvez également tester ces requêtes directement dans votre navigateur pour une vérification rapide du fonctionnement de notre API SMS. Il suffit de coller l’URL construite avec vos paramètres spécifiques dans la barre d’adresse et d’appuyer sur Entrée.

https://www.winsmspro.com/sms/sms/api?action=send-sms&api_key=VotreCléAPI&to=216xxxxxxxx&from=VotreIDExpéditeur&sms=VotreMessage

C’est une méthode pratique pour tester l’API WinSMS sans même avoir besoin de rédiger ou d’exécuter du code, ce qui peut être particulièrement utile pour les développeurs en train de se familiariser avec notre API ou pour ceux qui cherchent à effectuer des tests rapides d’envois d’SMS par API en Tunisie

  hostname: 'www.winsmspro.com',

  method: 'GET',
  headers: {
       'Content-Type': 'application/x-www-form-urlencoded',

3- Envoi d’un SMS à un seul numéro avec attente de la réponse du serveur en Node.js

Lorsque vous avez besoin d’une confirmation pour vos SMS envoyés de la part du serveur WinSMS, ce code Node.js vous aidera à l’obtenir tout en permettant un délai raisonnable pour la réponse.

NOS PRIX ET NOS PLANS

const https = require('https');

// Remplacez par votre clé API réelle que vous trouverez dans votre Espace Client WinSMS.tn.
const apiKey = 'votre_clé_api';

// Endpoint de l'API pour consulter le solde.
const url = `https://www.winsmspro.com/sms/sms/api?action=check-balance&api_key=${apiKey}&response=json`;

// Options pour la requête HTTPS.
const options = new URL(url);

// Initialisation et exécution de la requête pour consulter le solde.
https.get(options, (res) => {
  let data = '';
  
  // Récupération de la réponse.
  res.on('data', (chunk) => {
    data += chunk;
  });
  
  // Traitement de la réponse complète.
  res.on('end', () => {
    console.log("Réponse de l'API : " + data);
  });
  
}).on('error', (e) => {
  console.error(`Erreur lors de l'exécution de la requête : ${e.message}`);
});

5- Consultez Votre Solde WinSMS avec Notre API en Node.js

Pour nos clients désirant une gestion efficace de leur comptes SMS en Tunisie dans leurs différentes solutions, WinSMS met à disposition une fonctionnalité pratique via notre API SMS : la consultation de solde directement via une URI ou un script Node.js.

Cet exemple de code simplifié vous permet d’interroger rapidement votre solde disponible, vous assurant ainsi une transparence totale et une maîtrise complète de vos ressources SMS. Suivez ces étapes pour intégrer cette fonctionnalité  à vos solution internes de gestion SMS.

NOS PRIX ET NOS PLANS

const https = require('https');

// Remplacez par votre clé API réelle que vous trouverez dans votre Espace Client WinSMS.tn.
const apiKey = 'votre_clé_api';

// URL de l'API pour consulter le solde, incluant la clé API et le format de réponse souhaité.
const url = `https://www.winsmspro.com/sms/sms/api?action=check-balance&api_key=${apiKey}&response=json`;

// Initialise et exécute la requête HTTPS pour consulter le solde.
https.get(url, (res) => {
    let data = '';
    console.log('Statut HTTP:', res.statusCode);

    res.on('data', (chunk) => {
        data += chunk;
    });

    res.on('end', () => {
        // Affiche la réponse de l'API. Utile pour vérifier le solde disponible.
        console.log("Réponse de l'API :", data);
    });
}).on('error', (e) => {
    // Gère l'erreur, si elle se produit.
    console.error('Erreur lors de la requête :', e);
});

Ce code Node.js utilise le module https natif pour effectuer une requête GET simple, afin de consulter le solde via l’API WinSMS.

6- Vérification de l’État d’Envoi d’un SMS avec l’API WinSMS (Statut) en Node.js

Après avoir envoyé un SMS via notre API, il est essentiel de pouvoir vérifier l’état de cet envoi pour s’assurer qu’il a été correctement traité. Que vous souhaitiez confirmer la livraison ou diagnostiquer des problèmes éventuels, l’API WinSMS fournit des réponses claires et concises. Voyons ensemble comment implémenter une telle vérification avec Node.js.

NOS PRIX ET NOS PLANS

const https = require('https');
const querystring = require('querystring');

// Remplacez par votre clé API réelle que vous trouverez dans votre Espace Client WinSMS.tn.
const apiKey = 'votre_clé_api';

// La référence du SMS retournée par l'API lors de l'envoi.
const ref = '123456789';

// Construction des paramètres de la requête.
const params = querystring.stringify({
  'api_key': apiKey,
  'ref': ref // Utilisez la référence récupérée de votre envoi d'SMS par API.
});

// Endpoint de l'API pour consulter l'état d'un SMS.
const url = `https://www.winsmspro.com/sms/sms/api/verifSms?${params}`;

// Initialise et exécute la requête HTTPS pour vérifier l'état du SMS.
https.get(url, (res) => {
    let data = '';
    console.log('Statut HTTP:', res.statusCode);

    res.on('data', (chunk) => {
        data += chunk;
    });

    res.on('end', () => {
        // La requête a réussi. Traitement ou affichage de la réponse pour connaître l'état du SMS.
        console.log("Statut de l'envoi :", data);
    });
}).on('error', (e) => {
    // Gère l'erreur, si elle se produit.
    console.error('Erreur lors de la requête :', e);
});

Ce snippet en Node.js utilise les modules https et querystring pour réaliser une requête GET, afin de consulter l’état d’un SMS envoyé.

Astuce pour les développeurs : Vous pouvez également tester ces requêtes directement dans votre navigateur pour une vérification rapide du fonctionnement de notre API SMS. Il suffit de coller l’URL construite avec vos paramètres spécifiques dans la barre d’adresse et d’appuyer sur Entrée.

https://www.winsmspro.com/sms/sms/api/verifSms?api_key=VOTRE-API-KEY&ref=123456789
